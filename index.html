ian.js

ian.js is a lightweight framework built with priorities on common tasks and alleviate needless repition. The logic and methods are ment to feel native and barebones while powerful built-in functionality centers on dynamic content and simple, feature-rich DOM manipulation.

Consider appending a section of content to a news feed in pure javascript.

var item = document.createElement('div');
item.className = 'news-feed-item';

var img = document.createElement('img');
img.src = '/img/my-image.png';
item.appendChild(img);

var h1 = document.createElement('h1');
h1.textContent = 'My Title';
item.appendChild(h1);

var p = document.createElement('p');
p.textContent = 'A really long story.';
item.appendChild(p);

document.getElementById('news-feed').appendChild(item);

One of the main motivations for this library is allowing single-line implementations including chaining when available. 

var item = $('news-feed').append('div', { className: 'news-feed-item' });

item.append('img', { src: '/img/my-image.png' });
item.append('h1',  { textContent: 'My Title' });
item.append('p',   { textContent: 'A really long story.' }):

Notice that query-specific language is not being used like querySelector or the familiar jQuery library. The following example explains how querying with different identifiers can be done. 

var elFromId     = $('id');
var elsFromClass = $.class('a-class');
var elsFromTag   = $.tag('pre');
var elsFromName  = $.name('a_name');
var elsFromQuery = $.query('input[type=button]');

To further elaborate, the following is an example of chaining to get an existing element, change its properties, and then append a new child element. 

$('container').class('item')[0].set({
    style: {
        backgroundColor: '#abcdef';
    },
    onclick: function () {
        this.classList.toggle('selected-item');
    }
}).append('input', {
    type: 'button',
    className: 'close-button',
    onclick: function () {
        this.parentElement.goAway();
    }
});

ian.ajax.js

XMLHttpRequest interface for making and handling server requests/responses.
Dependency: 

ian.js

The core, light-weight library also focuses on only loading essential logic while allowing flexibility to advanced tasks through extensibility. Here, ian.ajax.js wraps up AJAX requests using the XMLHttpRequest API into a high-level plug-in. 

$.ajax.post('/my/sever-side/script.php', {
    data: {
        foo: 'lorem',
        bar: 'ipsum'
    },
    onload: function () {
        if (!this.response.error) hooray();
    }
});

messagebox.js

A customizable yet robust message box platform is necessary for any full-featured website or webapp. The messagebox.js library provides a complete interface between the message box DOM and the front-end logic including templating for complex markups and asynchronous callbacks.

A simple use case may involve displaying a message to the user or asking the user to provide a simple response. The 

messagebox
function takes either a string or an HTML element as its content argument, followed by an optional array of strings for button options. 

messagebox('Hello World!', ['Hello', 'Goodbye']);

Because the messagebox function is asynchronous, no value is returned. The next example shows how to handle a user's response using the callback function argument. 

messagebox('You have un-saved changes, are you sure you want to continue?',
    ['Yes', 'Cancel'],
    function (response) {
        if (response === 'Yes')
            continueToAnotherThing();
    }
);

For a final example, when utilizing the messagebox library's templating feature, a form element can be created in the HTML source, and shown by the message box the be filled and submitted. In general, multiple messagebox tempalte containers can be added to a page using the 

-messagebox-templates
class which is useful when generating dynamic content. Each child element in a template container is added to the template array to be accessed by its id attribute or data-name attribute if registering an id to the DOM is not desired.

Aside: regarding the example logic, the FormElement.submit() method does not raise the onsubmit event or trigger form validation and can be substituted with a visible submit button, or a hidden submit button invoked in your source by the messagebox callback.

<div class="-messagebox-templates"> <form id="my-form" action="submit.php"> <table> <tr> <td>First Name</td> <td><input type="text" name="first_name"></td> </tr> <tr> <td>Last Name</td> <td><input type="text" name="last_name"></td> </tr> </table> </div>
messagebox('my-form', ['Submit', 'Cancel'], function (response, form) { if (response !== 'Submit') return; // handle something form.submit(); });

As one final note, it is important to understand how the messagebox can be dismissed. The first of three methods is after clicking any button defined by the messagebox button argument array. As expected, this means any button input defined in the content section does not inherently invoke the callback function or close the message box. The second method is by calling the messagebox.close() method, which skips any passed callback function. The third method is by simply clicking anywhere on the messagebox mask covers the screen.
